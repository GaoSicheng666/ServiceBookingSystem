> 面向:运维/后端同学。记录后端服务在阿里云 Windows 服务器上的日常操作。
> 服务器公网 IP:`<公网IP>`  |  服务端口:8080
> 敏感凭据见同目录《部署凭据与配置.md》。
> 最后更新:2026-07-10

## 0. 如何连上服务器

**方式一:远程桌面(推荐,操作方便)**
- 本机 `Win + R` → `mstsc` → 填公网 IP `<公网IP>` → 用户名 `Administrator` → 密码
- 传文件:mstsc 连接前,「显示选项 → 本地资源 → 详细信息 → 勾选驱动器」,连进去后在"此电脑"里能看到本机磁盘,直接拷贝

**方式二:阿里云 ECS Workbench(网页版)**
- 阿里云控制台 → 实例 → 远程连接 → Workbench

> ⚠️ 操作完可以直接关窗口(=断开,服务照常运行),但**不要点"注销"**——注销会话会导致手动启动的服务停止(除非已注册为 Windows 服务)。

## 1. 启动后端服务

### 方式 A:手动启动(临时/调试用,窗口关了就停)

在服务器上开 `cmd`,逐条执行:
```cmd
set SERVICE_DB_USER=booking_app
set SERVICE_DB_PASSWORD=<数据库专用账号密码>
set SERVICE_JWT_SECRET=<JWT密钥,≥32字节随机串>
cd C:\booking
java -jar service-booking-backend-1.0.0.jar
```
看到 `Tomcat started on port 8080` + `Started ServiceBookingBackendApplication` 即成功。
**此窗口不能关**,关了服务就停。

### 方式 B:注册为 Windows 服务(推荐,开机自启、后台常驻)

见第 5 节。注册后用下面命令控制:
```cmd
net start BookingBackend      REM 启动
net stop  BookingBackend      REM 停止
```

---

## 2. 停止后端服务

- 手动启动的:在运行的 cmd 窗口按 `Ctrl + C`,或直接关窗口。
- 注册成服务的:`net stop BookingBackend`
- 强制杀掉(查不到窗口时):
  ```cmd
  REM 找到占用 8080 的进程号
  netstat -ano | findstr :8080
  REM 用上一步查到的 PID 结束进程
  taskkill /PID <进程号> /F
  ```

## 3. 更新后端(部署新版本 jar)

1. 本机重新打包:在 `D:\ClaudeWorkspace5\ServiceBookingBackend` 执行 `mvn package -DskipTests`,得到 `target\service-booking-backend-1.0.0.jar`

3. 把新 jar 传到服务器,覆盖 `C:\booking\service-booking-backend-1.0.0.jar`
4. 重新启动(见第 1 节)

## 4. 验证服务是否正常

**浏览器**(任意电脑)访问:
```
http://<公网IP>:8080/api/services
```
- 返回 `{"code":401,"message":"未登录或令牌缺失",...}` → 服务正常(401 是因为没带令牌,属正常)
- 打不开/超时 → 服务没启动,或安全组 8080 未放行,或 MySQL 没起

**完整业务验证**(PowerShell,逐行执行,IP 换成实际值):
```powershell
# 管理员登录拿 token(token 在 data.token 里)
$r = Invoke-RestMethod "http://<公网IP>:8080/api/auth/login" -Method Post -ContentType "application/json" -Body '{"username":"admin","password":"<管理员密码>"}'
$r.data.token
```
> 提示:PowerShell 显示中文乱码是终端编码问题,不影响功能;认 `code=0` 即成功。
> 可先执行 `[Console]::OutputEncoding=[System.Text.Encoding]::UTF8` 让中文正常显示。

## 5. 把后端注册为 Windows 服务(nssm)—— 开机自启

让服务后台常驻、开机自动运行,不再依赖 cmd 窗口。

### 5.1 下载 nssm
- 服务器浏览器访问 https://nssm.cc/download,下载 nssm 压缩包
- 解压,把 `win64\nssm.exe` 复制到 `C:\booking\`

### 5.2 安装服务
以管理员开 `cmd`,执行:
```cmd
cd C:\booking
nssm install BookingBackend
```
弹出图形界面,填写:
- **Application 标签**
  - Path: `C:\Program Files\Zulu\zulu-25\bin\java.exe`(你的 JDK 路径)
  - Startup directory: `C:\booking`
  - Arguments: `-jar service-booking-backend-1.0.0.jar`
- **Environment 标签**(每行一个,填入环境变量):
  ```
  SERVICE_DB_USER=booking_app
  SERVICE_DB_PASSWORD=<数据库专用账号密码>
  SERVICE_JWT_SECRET=<JWT密钥,≥32字节随机串>
  ```
- 点 **Install service**

### 5.3 启动服务
```cmd
net start BookingBackend
```
之后服务器每次开机都会自动运行,可以放心关远程桌面。

### 5.4 查看日志(可选)
可在 nssm 的 I/O 标签里配置 stdout/stderr 输出到文件,如 `C:\booking\log.txt`,便于排查。

### 5.5 卸载服务(需要时)
```cmd
nssm remove BookingBackend confirm
```

## 6. 常见故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 浏览器访问超时 | 服务没启动 / 8080 没放行 | 检查服务是否运行;阿里云安全组放行 8080 |
| 启动报 `Access denied for user 'booking_app'` | 数据库密码错 | 核对 `SERVICE_DB_PASSWORD` 与实际一致 |
| 启动报 `Unknown database` | 库没建 | MySQL 里执行 `CREATE DATABASE service_booking_system DEFAULT CHARSET utf8mb4;` |
| 启动报 `Communications link failure` | MySQL 服务没起 | `net start MySQL80` 启动 MySQL |
| 启动报 8080 被占用 | 上个进程没退 | 见第 2 节强制杀进程 |
| 接口返回 401 | 没带令牌或令牌过期 | 重新登录拿 token,请求头加 `Authorization: Bearer <token>` |
| 接口返回 500 | 服务器内部错误 | 看运行窗口/日志文件的红色堆栈,定位 `Exception`/`Caused by` |
| 中文显示乱码 | 终端编码 | 仅显示问题,不影响数据;PowerShell 可设 UTF8 输出 |

## 7. MySQL 常用操作

登录:
```cmd
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql" -u root -p
```
常用查询:
```sql
USE service_booking_system;
SHOW TABLES;                    -- 查看所有表
SELECT * FROM users;            -- 所有用户
SELECT * FROM employees;        -- 所有员工
SELECT * FROM appointments;     -- 所有预约单
-- 谁约了谁(把 id 翻译成姓名)
SELECT u.name 用户, e.name 员工, s.name 服务, a.appointment_date 日期, a.status 状态
FROM appointments a
JOIN users u ON a.user_id=u.id
JOIN employees e ON a.employee_id=e.id
JOIN services s ON a.service_id=s.id;
```

MySQL 服务控制:
```cmd
net start MySQL80      REM 启动
net stop  MySQL80      REM 停止
```
