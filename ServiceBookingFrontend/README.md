# 老年人服务预约系统前端

这是一个原生 HTML/CSS/JavaScript 单页前端，用于对接 `ServiceBookingBackend` 的 Spring Boot REST API。

## 使用方式

直接打开：

```text
ServiceBookingFrontend/index.html
```

后端地址固定配置在 `app.js` 中：

```text
http://121.40.96.66:8080/api
```

## 前端不需要 MySQL

本前端只通过 HTTP/JSON 调用后端接口，不连接 MySQL。MySQL 只由后端服务器使用。

## 后端接口约定

登录成功后保存后端返回的 JWT token。除注册和登录外，请求都会自动携带：

```text
Authorization: Bearer <token>
```

同一账号只允许一个有效登录会话。账号在另一台设备或另一个独立浏览器中重新登录后，旧会话下一次请求接口时会自动退出并提示重新登录。

## 护工培训流程

1. 新注册护工首次登录后进入“护工服务须知”学习页。
2. 护工确认完成学习，或管理员在员工管理中点击“开放答题”。
3. 护工完成 4 道单选题，答对至少 3 题后直接进入工作台。
4. 工作台内可以随时设置星期一至星期日的 `06:00-10:00`、`10:00-14:00`、`14:00-18:00`、`18:00-22:00` 可工作时段。
5. 培训未通过前不能开启接单，也不会出现在老人端的可预约护工列表中。

培训状态、答题结果和工作时段都保存在后端 MySQL 中，不是浏览器临时数据。

## 护工个人信息

护工通过培训进入工作台后，可以在“个人信息”中修改真实姓名、年龄、联系电话、从业经历和个人简介，并上传或移除头像。“可胜任服务”位于工作页的每周工作时间表下方，不是自由输入文字，而是从管理员当前上架的服务项目中多选并单独保存；老人选择某项服务时只会看到已选择该服务、当天有空且正在接单的护工。浏览器会先把头像压缩到最长边 512 像素，再提交后端保存。

个人信息发生修改但尚未保存时，切换到工作任务、刷新、退出或关闭页面都会先提示确认。保存每周工作时间、切换接单状态、完成服务和取消任务等重要操作也会进行二次确认。

## 老人预约与金额规则

1. 先选择服务项目，再选择服务日期。
2. 系统列出当天至少还有一个空闲排班时段的护工，老人可以展开查看护工资料后确认选择。
3. 选定护工后，系统显示该护工当天剩余的 `06:00-10:00`、`10:00-14:00`、`14:00-18:00`、`18:00-22:00` 时段，老人可以选择一个或多个。
4. 预约总额为“服务参考单价 × 所选时段数”，确认页会在提交前显示计算过程。
5. 同一老人或护工在同一天可以有多条预约，但时段不能重叠；提交预约时后端会再次检查排班和冲突。

> 本功能依赖最新的 `ServiceBookingBackend`。只更新前端文件无法生效，还需要重新打包并部署后端 JAR；后端重启时会自动创建新增数据表。
