# 老年人服务预约系统前端

这是一个原生 HTML/CSS/JavaScript 单页前端，用于对接 `ServiceBookingBackend` 的 Spring Boot REST API。

## 使用方式

直接打开：

```text
ServiceBookingFrontend/index.html
```

后端地址固定配置在 `app.js` 中：

```text
http://121.40.96.66:8080/api
```

## 前端不需要 MySQL

本前端只通过 HTTP/JSON 调用后端接口，不连接 MySQL。MySQL 只由后端服务器使用。

## 已覆盖页面

- 登录
- 注册老人用户
- 注册护工员工
- 老人端：适老化服务首页、四步预约向导、日期与四时段多选、按时段数计算金额、卡片式预约记录、查看和取消预约
- 员工端：岗前学习、4 道安全常识题、周一至周日四时段工作时间表、已获得薪资总额、今日任务工作台、个人信息与头像维护、接单状态开关、弹窗查看联系电话、高德导航、完成任务、填写原因后取消任务
- 管理员端：服务项目管理、用户管理、员工培训状态与开放答题权限、预约总览
- 品牌图标：页面左上角与浏览器标签页使用 `assets/app-icon.png`
- 登录页插画：未登录时使用 `assets/auth-care-illustration.png` 展示独立欢迎页

## 后端接口约定

登录成功后保存后端返回的 JWT token。除注册和登录外，请求都会自动携带：

```text
Authorization: Bearer <token>
```

同一账号只允许一个有效登录会话。账号在另一台设备或另一个独立浏览器中重新登录后，旧会话下一次请求接口时会自动退出并提示重新登录。

## 护工培训流程

1. 新注册护工首次登录后进入“护工服务须知”学习页。
2. 护工确认完成学习，或管理员在员工管理中点击“开放答题”。
3. 护工完成 4 道单选题，答对至少 3 题后直接进入工作台。
4. 工作台内可以随时设置星期一至星期日的 `06:00-10:00`、`10:00-14:00`、`14:00-18:00`、`18:00-22:00` 可工作时段。
5. 培训未通过前不能开启接单，也不会出现在老人端的可预约护工列表中。

培训状态、答题结果和工作时段都保存在后端 MySQL 中，不是浏览器临时数据。

## 护工个人信息

护工通过培训进入工作台后，可以在“个人信息”中修改真实姓名、年龄、联系电话、擅长服务、从业经历和个人简介，并上传或移除头像。浏览器会先把头像压缩到最长边 512 像素，再提交后端保存；老人选择服务人员时可以看到这些公开资料。

## 老人预约与金额规则

1. 先选择服务项目，再选择服务日期。
2. 在 `06:00-10:00`、`10:00-14:00`、`14:00-18:00`、`18:00-22:00` 中选择一个或多个时段。
3. 系统只显示在所选星期和全部所选时段都有排班、并且尚未被预约的护工。
4. 预约总额为“服务参考单价 × 所选时段数”，确认页会在提交前显示计算过程。
5. 同一老人或护工在同一天可以有多条预约，但时段不能重叠。

> 本功能依赖最新的 `ServiceBookingBackend`。只更新前端文件无法生效，还需要重新打包并部署后端 JAR；后端重启时会自动创建新增数据表。
